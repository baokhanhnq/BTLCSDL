# Git workflow for BTLCSDL

## What to upload

Commit source code and project configuration:

- `ASW/`: application logic.
- `BSW_ECUAL/`: drivers for sensors, motor, throttle, alerts.
- `RTE/`: shared runtime interface between drivers and application logic.
- `Config/`: system configuration and thresholds.
- `Core/`: STM32Cube generated and user source files.
- `Drivers/`: STM32 HAL/CMSIS drivers generated by CubeMX.
- `BTLCSDL.ioc`: pin and peripheral configuration.
- `.project`, `.cproject`, `.mxproject`: STM32CubeIDE project metadata.
- `STM32F401RETX_FLASH.ld`, `STM32F401RETX_RAM.ld`: linker scripts.

Do not commit build output or local workspace files:

- `Debug/`
- `Release/`
- `.settings/`
- `.codex-build-workspace/`
- `*.elf`, `*.hex`, `*.bin`, `*.map`, `*.list`, `*.o`, `*.d`, `*.su`
- `*.launch`

## Suggested team split

### Member 1: ASW logic

Main files:

- `ASW/App_AebLogic.c`
- `ASW/App_AebLogic.h`
- `ASW/Filter_Median.c`
- `ASW/Filter_Median.h`

Responsibility:

- AEB and FCW state machine.
- Distance thresholds by speed.
- Median filter behavior.
- Safety release condition.

### Member 2: Sensors and input

Main files:

- `BSW_ECUAL/Sensors/HCSR04.c`
- `BSW_ECUAL/Sensors/HCSR04.h`
- `BSW_ECUAL/HMI/Throttle.c`
- `BSW_ECUAL/HMI/Throttle.h`
- `Core/Src/adc.c`
- `Core/Inc/adc.h`
- `Core/Src/delay.c`
- `Core/Inc/delay.h`

Responsibility:

- HC-SR04 distance measurement.
- Throttle ADC reading and motor speed command.
- Microsecond/millisecond delay.
- Sensor calibration.

### Member 3: Motor and alerts

Main files:

- `BSW_ECUAL/Motor/L298N_Driver.c`
- `BSW_ECUAL/Motor/L298N_Driver.h`
- `BSW_ECUAL/HMI/Alerts.c`
- `BSW_ECUAL/HMI/Alerts.h`

Responsibility:

- L298N motor direction and PWM.
- Brake behavior.
- Green/yellow/red LEDs.
- Buzzer blinking/solid behavior.

### Member 4: Integration and configuration

Main files:

- `RTE/Rte.c`
- `RTE/Rte.h`
- `Config/system_config.h`
- `Core/Src/main.c`
- `Core/Src/uart.c`
- `Core/Inc/uart.h`
- `BTLCSDL.ioc`

Responsibility:

- Main loop integration.
- RTE data flow.
- UART logging.
- Pin configuration.
- Build and final hardware test.

## Branch rule

Use one branch per task:

- `feature/asw-aeb-logic`
- `feature/sensor-hcsr04`
- `feature/motor-alerts`
- `feature/rte-integration`

Merge through pull requests. The leader should review every pull request before merging into `main`.

## Commit examples

Use short, clear commit messages:

- `Add HC-SR04 cm distance reading`
- `Update AEB thresholds by PWM speed`
- `Fix motor stop behavior below 40 percent`
- `Add UART status logging`
